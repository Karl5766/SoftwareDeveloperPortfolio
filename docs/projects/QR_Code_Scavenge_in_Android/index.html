QR Code Scavenge in Android

Commonly, software developers work in teams with Agile methodologies, which is a set of methods that promote regular progress meetings, frequent pull requests, and adaptability to changes.

We developed an Android app named QRCode-Quest in an Agile team of 6 as part of my undergraduate course. The app is built using Android Studio as the IDE and depends on the following components:

  • Android Navigation
  • OpenStreetMap
  • Firebase Firestore and Storage
  • ZXing for QR code scanning
  • JUnit
  • Espresso
  • Mockito

My responsibilities in this team included developing and adapting the Firebase database to integrate with the rest of the app and organizing daily meetings before demo delivery. As with the rest of the team, I collaborated on planning the project, including drawing CRC cards and UML diagrams, and demonstrating the delivered app to the class.

App Screenshots

Below are a few screenshots of our app:

The login page is one where the users create their accounts and login using the QR code scanning functionality. This page re-uses the QR scanning functionality to remove the need for repeatedly entering the passwords during login and hashes the passwords for security. A remember-username feature is also added for convenience.

QR code capture page allows the user to scan QR pages to gain points and share the scanned codes for other people. In this project we are working with a virtual camera in a simulated environment so we are able to create unit tests for these cases without the need to physically connect to a phone or camera every time the unit tests are run. This improves the efficiency of CI/CD and allows quick and reliable testing of the QR code feature

The ranking page sorts scores by some order and display the ranking that other users obtained. Users that go around to collect QR code can compare the total scores they get against others'.

In-app map allows users to find QR codes captured by other people and go take a photo themselves. This functionality is implemented with OpenStreetMap, a popular open-source alternative to Google Map

As the project progressed, we gained significant hands-on experience with Android Studio and its UI and testing frameworks. By the end of the project, I learned a lot on software development processes, teamworks and Agile practices, and contributed over 6000 lines of changes to the repository, and our team worked together to ensure on-time delivery and a successful demo of the app.
